# Changelog

## 4.2.0 - 2026-03-25

### 版本概览
本次 4.2.0 重点补强 `window.__adapter_zip__` 的多段拼接解析，修复 adapterZip 资源被拆成多段 `+=` 时无法完整识别的问题。

### 更新内容
- 修复 `window.__adapter_zip__` 被分成 `=` + 多个 `+=` 段时，只读取首段导致解压失败的问题。
- 优化 zip / adapterZip 提取逻辑：从首段赋值开始，继续全局收集同变量后续拼接段。
- 修复部分 Cocos playable 明明包含完整资源表，但工具提示无法解析或仅识别极少图片的问题。
- 保持 4.1.0 中 `script[type="text/base64"]`、zip 文本内嵌图扫描与回写逻辑不变。

## 4.1.0 - 2026-03-24

### 版本概览
本次 4.1.0 重点修复 super-html 变体解析缺口，补齐 `script[type="text/base64"]` 入口和 zip 内文本文件中的内嵌图片识别。

### 更新内容
- 新增对 `script[type="text/base64"]` 形式 zip 入口的识别，不再仅依赖 `window.__zip` / `+=` 赋值。
- 新增对 zip 内 `js/json/css/html/xml/txt` 等文本条目中 `data:image` 资源的扫描与展示。
- 修复部分 playable 资源全部内嵌在 `2.js` 等脚本文件时，图片列表只能识别出 1 张的问题。
- 支持将替换后的内嵌图片回写到 zip 内原始文本条目，保持生成结果可用。
- 优化 super-html 解析状态文案，显示具体入口来源与解析类型，便于排查。

## 4.0.0 - 2026-03-16

### 版本概览
本次 4.0.0 重点从“替换工具”升级为“替换 + 预览”一体化工作台，补齐本地预览、设备比例切换与桌面端操作布局。

### 更新内容
- 新增本地 iframe 预览，生成后可直接在面板内查看 playable 结果。
- 新增多设备预设与横竖屏切换，支持 iPhone、iPad、Pixel、Samsung 等常见尺寸。
- 重构主界面为三栏布局，分别承载说明、替换操作和设备预览。
- 上传 HTML 后自动解析资源，不再需要单独点击“解析资源”。
- 优化中间图片资源列表与右侧预览区的高度分配，减少无效留白。

## 3.1.0 - 2026-03-16

### 版本概览
本次 3.1.0 重点补强了 `window.__zip` 回写后的交付稳定性，解决换图或改链接后文件体积明显变大的问题。

### 更新内容
- 新增 `__zip` 高压缩回写，重打包时显式使用 `DEFLATE`。
- 修复部分 playable 在改动后 `__zip` 压缩率下降，导致 HTML 从 4MB 膨胀到 7MB 左右的问题。
- 保持现有 `window.__zip`、`window.__res`、批量换图与跳转重写逻辑不变。

## 2.0.0 - 2026-03-13

### 版本概览
本次 2.0.0 主要围绕高频调试同一个 playable 的使用场景进行了优化，重点提升稳定性和换图效率。

### 更新内容
- 修复了连续生成 HTML 后文件体积不断变大的问题。
- 修复了反复调试时修改内容层层叠加、结果越来越不稳定的问题。
- 新增批量导入图片功能，支持一次选择多张图片自动匹配替换。
- 新增批量替换结果提示，可直接查看已匹配和未匹配数量。
- 对不适合批量替换的模式增加了明确提示，减少误操作。

### 使用体验提升
- 更适合反复导入、反复改图、反复生成的工作流程。
- 多图替换效率更高，减少逐张上传的重复操作。
- 生成结果更干净，历史残留更少，更方便检查和交付。
